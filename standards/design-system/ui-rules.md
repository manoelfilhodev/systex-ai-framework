# Regras de UI

## Diretrizes obrigatorias

- Dark theme e o padrao inicial para produtos Systex.
- A tela deve priorizar tarefa, decisao e leitura de informacao.
- Componentes interativos devem ter estados de hover, focus, active, disabled, loading e error.
- Acoes destrutivas devem ser visualmente diferenciadas e exigir confirmacao quando houver risco.
- Formularios devem ter labels claros, validacao de entrada e mensagens de erro objetivas.
- Layouts devem ser responsivos e testados em desktop e mobile quando aplicavel.
- Tabelas devem ter alinhamento, filtros, ordenacao ou paginacao quando o volume exigir.
- Empty states devem orientar o proximo passo do usuario.
- Loading states devem evitar telas congeladas ou ambiguas.

## Nao permitido

- Criar interface sem hierarquia visual.
- Misturar muitas cores de acento.
- Usar White Label sem aprovacao de escopo.
- Esconder erro tecnico sem log ou mensagem adequada.
- Criar componentes visuais fora do padrao sem justificativa.
