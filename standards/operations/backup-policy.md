# Politica de backup

## Escopo

Backup deve cobrir banco de dados, arquivos relevantes, configuracoes criticas e assets de cliente quando aplicavel.

## Banco de dados

- Realizar backup antes de alteracoes criticas.
- Definir periodicidade conforme risco e contrato.
- Proteger arquivos de backup contra acesso indevido.

## Arquivos

- Incluir uploads, documentos, imagens e arquivos operacionais relevantes.
- Separar arquivos temporarios de arquivos permanentes quando possivel.

## Retencao

A retencao deve ser definida conforme contrato, criticidade do projeto e custo operacional.

## Restauracao

Backup sem restauracao testada nao deve ser considerado garantia operacional. Projetos criticos devem prever teste periodico de restauracao.

## Responsabilidade

A responsabilidade entre cliente e Systex deve ser definida em contrato, incluindo frequencia, armazenamento, custo, retencao, restauracao e resposta a incidentes.
